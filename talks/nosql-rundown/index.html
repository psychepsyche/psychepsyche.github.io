NoSQL Rundown

March 24, 2011

Goal

Let's discuss how you might use new tech to complement a relational database

What is NoSQL?

  • Document stores
  • Graph stores
  • Key-value stores

  • JSON-like documents
  • Index any field
  • Rich queries

If it makes sense for your model

class User()
    string Name
    List<string> Addresses
    List<string> PhoneNums
    List<Order> Orders

class Order()
    int Price
    List<Widget> Items

Relational model starts to get bloated

tblUsers
    -tblUserAddresses
    -tblUserPhoneNums
    -tblUserOrders
tblOrders
    -tblOrderItems

Join, Join, Join ...

{
    "name":"matt",
    "addresses": [
        "123 Main St", "4 Center Green"
    ],
    ...

    "orders": [
        {
            "name": "Learn MongoDB",
            "price": 30
        },
        {
            "name": "Widget #7",
            "price": 77
        }
    ]
}
db.Users.find(
    {"orders.name" : "Learn MongoDB"}
)

vs

select * from tblUsers U
join tblUserOrders UO
on UO.UserId = U.oid
join tblOrders O
on O.oid = UO.OrderId
where O.Name = "Learn MongoDB"

Real-time Analytics

  • Upset + $inc
  • Fire and forget
  • Write heavy
  • Flexible reporting

  • Stores graph data (edges, vertices)
  • Both can have properties
  • Handles user-generated, recursive data well

http://wiki.neo4j.org/content/The_Matrix

Use cases?

  • Network oriented data
  • Hop = Join
  • Social Graph
  • Spatial Graph

  • In memory, key-value* store
  • Non-blocking, single threaded
  • 100k reads/writes per sec

Swiss army knife for data

  • Binary-safe strings
  • List, set, sorted set, hash
  • Pub/sub channels
  • Expirable keys

Caching

  • Hash input query
  • Shove result into redis
  • Set an expiration
  • Delete to invalidate

Web app sessions

  • Only accessed by primary key anyways
  • Lower per-page overhead
  • Expire automatically

tail logs

  • LPUSH + LTRIM into a list
  • Track latest N events

Task queues

  • LPOP/RPUSH on a list
  • Blocking atomic POPs for workers
  • Backend for resque and restmq

API Throttling

  • Store list using API key
  • Fast limit checks
  • Capped collections with expiration
